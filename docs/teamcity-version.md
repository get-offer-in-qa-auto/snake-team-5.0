# TeamCity Version Selection

Дата решения: 2026-06-29

## Выбранная версия

Для автотестов используем Docker-образ:

```bash
jetbrains/teamcity-server:2026.1.1
```

Если нужен явно Linux-образ, используем:

```bash
jetbrains/teamcity-server:2026.1.1-linux
```

## Почему выбрана версия 2026.1.1

Версия `2026.1.1` выбрана как целевая стабильная версия для нового тестового стенда, потому что это официальный bug-fix релиз ветки `2026.1`, опубликованный JetBrains 29 мая 2026 года.

Основные причины выбора:

- это закрепленная версия, поэтому автотесты будут воспроизводимыми;
- это не тег `latest`, который может измениться без изменений в нашем проекте;
- релиз `2026.1.1` исправляет проблемы, найденные после выхода `2026.1`;
- в релиз включены исправления багов, performance issues и security issues;
- ветка `2026.1` содержит важные изменения TeamCity, которые нужно учитывать в автотестах.

## Почему не используем latest

Тег `latest` на Docker Hub сейчас указывает на актуальный образ TeamCity, но для автотестов он не подходит как основной целевой тег.

Причина: `latest` может быть обновлен JetBrains в любой момент. Если тесты будут запускаться на `latest`, то при обновлении образа мы можем получить изменения поведения продукта без явного изменения конфигурации тестового проекта.

Поэтому для стабильного и воспроизводимого тестового окружения фиксируем конкретный тег:

```bash
jetbrains/teamcity-server:2026.1.1
```

## Что важно проверить в автотестах

При проектировании автотестов нужно учитывать изменения ветки `2026.1` и исправления `2026.1.1`.

Приоритетные зоны:

- старт сервера из Docker-образа;
- первичная настройка TeamCity;
- REST API;
- авторизация и работа с токенами;
- создание project и build configuration;
- запуск build и проверка статусов;
- подключение build agent;
- работа с Pipelines, если они входят в тестируемый scope;
- базовые security regression checks;
- проверка поведения после рестарта контейнера с сохраненным data directory.
